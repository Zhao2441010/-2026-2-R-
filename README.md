大二下软件工程小组作业
前端Vue,vue容器预写,编译
后端springboot
工具链:maven管理,mysql,loombook,jpa,等

本项目实现了对流浪猫信息录入,照片上传,下发志愿任务与报名,用户建议上报与回复功能
用户访问猫信息,上传近期照片,提交信息

接入Kimi,实现对用户简单问题回答与图片处理
支持用户上传照片查询猫的种类与根据ai判断疾病大致类型的功能



Spring 容器：所有用到的实例预创建成 Bean 放在容器内，随用随取，避免每个实例的单独创建和复杂创建依赖。
切片：一是将功能切成不同对象，面向对象编程。二是将功能分为 entity(数据库表)、repository(数据库操作)、service(可以调用多个repository来实现功能在数据库的修改)、controller(边界判断+调用多个service实现功能与返回内容) 等层次，方便 debug 等。
自动配置：约定优于配置，配置更简单，相关的配置代码更少。

Session：用户登录后服务端生成唯一 ID(也可以是对象)，后续请求带上这个 ID 就能识别是谁，不用每次都重新验证身份。
HTTP 请求运行流程：浏览器发请求 → Controller 接收并解析参数 → Service 处理业务逻辑 → Repository/Mapper 查数据库 → 数据原路返回 → 浏览器展示。各层各司其职，出了问题能快速定位到哪一层。

Debug ：图片上传默认存在项目文件里，导致重启造成用户退出,应该放在外部文件夹 等等一系列

防御性:
  1.注册时检测ip,避免一个ip多次注册
  2.限制每日上传数量,避免恶意攻击
  3.限制特殊字符输入比如`',避免sql注入

  可以添加:验证码,滑块验证
